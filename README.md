# 🚀 유튜브 소재 발굴 도구

**구독자 대비 조회수 비율(바이럴 비율)**로 강력한 콘텐츠 소재를 발굴하고, AI가 댓글을 분석해 대본 목차까지 자동 생성하는 웹 앱입니다. 더불어 AI 이미지/영상 생성, 한글 자막 굽기 등 콘텐츠 제작 기능을 제공합니다.

**📦 저장소**: https://github.com/david1005910/Youtube_matl_search_Mac1KHS
**📖 사용 설명서**: [MANUAL.md](MANUAL.md) — 검색부터 영상 생성까지 단계별 가이드

---

## ✨ 주요 기능

### 🔍 영상 검색 & 필터링
- 키워드 기반 YouTube 영상 검색 (한국어 우선)
- **영상 형태 필터**: 숏폼(3분 미만) / 롱폼 구간별 선택 (3~5분, 5~7분, 7~9분, 9~11분, 11~13분, 13~15분, 15분 이상)
- **최소 바이럴 비율 필터**: 50% ~ 5,000% 이상 선택 (기본값 200%)
- **검색 기간**: 최근 3개월 ~ 2년
- **결과 수**: 10 ~ 50개

### 📈 바이럴 비율 분석
> **바이럴 비율 = (조회수 ÷ 구독자 수) × 100**

구독자 규모와 무관하게 소재 자체의 폭발력을 측정합니다. 비율이 높을수록 알고리즘이 적극 추천한 영상입니다.

| 비율 | 의미 |
|------|------|
| 200% 이상 | 소재력 검증된 영상 |
| 500% 이상 | 강력한 바이럴 소재 |
| 1,000% 이상 | 알고리즘 폭발 영상 |

### 💬 AI 댓글 분석 (Gemini)
댓글 수집 버튼 클릭 한 번으로 자동 분석:

- ✅ **시청자 반응** — 열광하는 포인트
- ❓ **Pain Points** — 아쉬운 점 & 추가 궁금증
- 🏷️ **자주 언급된 키워드 TOP 5** — 언급 맥락 포함
- 🚀 **바이럴 3대 요소 분석** — CTR / AVD / 참여도 점수 & 개선 팁
- 🎬 **추천 주제 키워드 5개** — 기획 방향 + 예상 제목 포함

### ✍️ AI 대본 목차 자동 생성
추천 주제 중 1개 선택 → 분량 선택 → 대본 목차 생성

| 분량 | 내용 |
|------|------|
| 짧게 (~500자) | 핵심만 |
| 보통 (~1,000자) | 표준 구성 |
| 상세 (~2,000자) | 설명 포함 |
| 완전 대본 (~3,000자) | 실제 말할 수 있는 형식 |

생성 항목: 영상 제목 후보 3개 · 썸네일 컨셉 · 오프닝 훅 · 챕터별 대본 · 클로징 & CTA · 제작 팁

## 🛠️ 설치 및 실행

먼저 저장소를 클론합니다.

```bash
git clone https://github.com/david1005910/Youtube_matl_search_Mac1KHS.git
cd Youtube_matl_search_Mac1KHS
```

실행 방법은 두 가지입니다.

### 방법 1 — 로컬 서버 (권장)

`server.py`로 띄우면 API 키가 `.env` 파일에 저장되어 매번 다시 입력할 필요가 없고, AI 이미지/영상 생성 등 프록시 기능도 모두 동작합니다.

```bash
python3 server.py        # http://localhost:8765 접속
```

> macOS에서 SSL 인증서 오류가 나면 `pip3 install certifi` 후 다시 실행하세요.

### 방법 2 — 브라우저에서 바로 열기

별도 서버 없이 검색·분석·대본 기능만 빠르게 쓰고 싶을 때 사용합니다. (API 키는 브라우저 세션에만 저장되어 새로고침 시 다시 입력해야 합니다.)

```bash
open index.html          # 또는 index.html 파일을 더블클릭
```

### (선택) 자막 렌더러

영상에 자막을 굽는 Remotion 렌더러는 별도 서버로 실행합니다.

```bash
cd remotion && npm install && npm start   # http://localhost:8766
```

## 🎬 AI 영상 생성 (Phosphene · 로컬 MLX)

생성한 **카드 이미지를 영상(image→video)으로** 만드는 기능입니다. Apple Silicon 맥에서 로컬로 동작하는 **Phosphene**(LTX-2 모델, MLX)을 사용하며 — 클라우드도, API 키도 필요 없습니다.

### 준비
1. **Phosphene 설치·실행** — [Pinokio](https://pinokio.computer)에서 Phosphene을 설치하고 실행합니다. 패널이 `http://127.0.0.1:8198`에 뜹니다.
2. **`.env` 설정** (로컬 서버 모드) — 기본값으로 자동 연결됩니다. 포트가 다르면 지정하세요.
   ```
   PHOSPHENE_URL=http://127.0.0.1:8198
   ```
3. **(영상 합치기용) ffmpeg** — `brew install ffmpeg`

### 카드별 영상 생성
1. 이미지 카드를 생성합니다 (대본 → 이미지 생성).
2. 카드의 **`🎬 Phosphene 영상 생성`** 버튼을 누르면 그 이미지로 영상이 만들어집니다.
   - **⚙️ 옵션**: 해상도(736×416 ~ 1280×720 · 세로) · 길이(5/7/10초) · 품질(quick~high) · 모션 프롬프트(비우면 이미지 프롬프트 사용)
3. 완료되면 카드 안에서 영상을 바로 재생·다운로드할 수 있습니다.

### 전체 일괄 + 합치기
Grok 영상 위젯의 **`🎬 전체 카드 영상 생성 + 합치기 (Phosphene)`** 버튼을 누르면, 모든 카드를 순차로 영상화한 뒤 하나의 MP4로 이어붙여 내려받습니다.

> ⏱️ 로컬 GPU 렌더라 클립당 약 5분(736×416 기준), 한 번에 하나씩 순차 처리됩니다. 빠른 미리보기는 **736×416 / quick** 조합을 권장합니다.

### ✅ 검증 상태
실제 앱 화면에서 end-to-end로 동작을 확인했습니다 (2026-06-24):
- **카드별 영상 생성** — 이미지 업로드 → i2v 렌더 → 카드 안에서 영상 재생까지 정상
- **전체 일괄 + 합치기** — 카드 2개 순차 렌더 → ffmpeg로 하나의 MP4(10초)로 합쳐 다운로드까지 정상

## 🔑 API 키 설정

처음 실행 시 상단 패널에 두 가지 키를 입력합니다.

### YouTube Data API v3
1. [Google Cloud Console](https://console.cloud.google.com) 접속
2. 새 프로젝트 생성
3. **YouTube Data API v3** 활성화
4. 사용자 인증 정보 → API 키 발급

### Gemini API
1. [Google AI Studio](https://aistudio.google.com/app/apikey) 접속
2. **API 키 생성** (무료 티어 제공)

> **키 저장 방식**: 로컬 서버 모드(방법 1)에서는 키가 프로젝트 루트의 `.env` 파일에 저장됩니다(`.gitignore`로 커밋 제외). 브라우저 직접 실행 모드(방법 2)에서는 브라우저 세션에만 저장됩니다. 두 경우 모두 키는 본인 기기/서버에서만 사용됩니다.

## 💡 바이럴 비율 3대 핵심 요소

| 요소 | 설명 |
|------|------|
| 🎯 **CTR** (클릭률) | 썸네일·제목이 매력적일수록 바이럴의 시작점이 됩니다 |
| ⏱️ **AVD** (평균 시청 지속 시간) | 초반 30초가 핵심. 끝까지 볼수록 알고리즘이 추천합니다 |
| 💬 **Engagement** (참여도) | 좋아요·댓글·공유·구독 전환율이 알고리즘 점수를 높입니다 |
